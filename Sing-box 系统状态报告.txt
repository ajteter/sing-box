# Sing-box 系统状态报告

本报告旨在梳理当前 VPS 上 `sing-box` 的运行情况，并明确后续的稳定性保障方案。

---

## 1. 当前版本信息
经检测，当前系统部署的 `sing-box` 处于开发测试分支（Alpha），该版本虽然功能较新，但在稳定性上不及正式发布版本。

*   **当前版本**: `1.14.0-alpha.15`
*   **构建修订号**: `3367bde1390969ca0c7dee567478cb086cd97b12`
*   **部署方式**: 使用 `fscarmen/sing-box` 管理脚本安装
*   **运行状态**: 正常运行中 (Service active)

## 2. 状态评估
*   **稳定性**: **中等 (Alpha 版)**。Alpha 版本可能包含未修复的 Bug 或不稳定的 API 变更，不建议在生产环境长期运行。
*   **兼容性**: 与现有 `fscarmen` 脚本生态兼容，但升级策略需通过该脚本执行。
*   **更新来源**: GitHub `SagerNet/sing-box` (上游) -> `fscarmen` (本地安装管理脚本)

## 3. 切换稳定版的必要性
根据你制定的“仅限稳定版”原则，切换至 `Stable Release` 具有以下优势：
*   **可靠性提升**: 正式发布版经过广泛测试，适合长期稳定后台运行。
*   **可预测性**: 避免因 Alpha 版的频繁特性变动导致服务意外中断。
*   **监控对齐**: 与其他服务（如 `hermes-webui`）的稳定版管理规范保持一致。

## 4. 后续维护方案 (Action Plan)
为了平稳完成过渡并确保持续监控：

1.  **手动切换**: 执行 `bash /etc/sing-box/sb.sh` 并选择 **1 (安装/更新)**。脚本将识别环境并自动覆盖为当前官方正式稳定版。
2.  **配置留存**: 更新前，我会协助你进行配置文件的备份 (`config.json` -> `config.json.bak`)。
3.  **后续监控**: 在切换完成后，更新监控脚本将自动识别版本降级（回归稳定版），此后只会提醒你升级到更高的 **Stable** 版本，而不再提示降级或追逐 Alpha。
